# MLModels

Drop `CardioRiskPredictor.mlpackage` here (generated by `MLPipeline/prune_and_convert.py`).

Xcode 16's synchronized groups will pick it up automatically as part of the
`CardioGuard` target - no project file editing needed. `CoreMLCardioRiskPredictor`
loads it by name at runtime (`Bundle.main`) and fails soft if it isn't present,
so the app builds and runs without it; the AI early-warning card in the
dashboard simply stays hidden until the model is bundled.
